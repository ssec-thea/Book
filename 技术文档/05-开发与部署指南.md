# BookVoyage 开发与部署指南

## 1. 环境要求

| 软件 | 版本要求 | 说明 |
|------|---------|------|
| Node.js | ≥ 18.0 | JavaScript 运行时 |
| npm | ≥ 9.0 | 包管理器 (随 Node.js 安装) |
| MySQL | ≥ 8.0 | 数据库服务器 |
| Git | ≥ 2.0 | 版本控制 (可选) |

## 2. 快速开始

### 2.1 克隆项目

```bash
git clone https://gitee.com/ssecjyq/book.git
cd book/项目源码
```

### 2.2 安装依赖

```bash
npm install
```

### 2.3 配置环境变量

复制 `.env.example` 为 `.env` 并编辑：

```bash
cp .env.example .env
```

`.env` 文件内容：
```env
# Gemini AI API Key (可选, 不配置则使用本地回退)
GEMINI_API_KEY=your_gemini_api_key_here

# JWT 签名密钥 (生产环境请使用强随机字符串)
JWT_SECRET=your_jwt_secret_here

# MySQL 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=071231
DB_NAME=bookvoyage

# 服务端口
PORT=3000
```

### 2.4 初始化数据库

```bash
# 方式一: 使用 schema.sql 手动创建
mysql -u root -p < ../数据库表/schema.sql

# 方式二: 启动时自动创建 (推荐)
# 服务启动时会自动执行建表 + 种子数据
npm run seed   # 运行种子脚本
```

### 2.5 启动开发服务

```bash
npm run dev
```

访问 `http://localhost:3000` 查看应用。

## 5. 生产部署

### 5.1 构建

```bash
npm run build
```

此命令会：
1. `vite build` — 构建 React 前端到 `dist/`
2. `esbuild server.ts --bundle --platform=node --outfile=dist/server.cjs` — 编译服务端

### 5.2 启动

```bash
NODE_ENV=production npm start
```

生产模式下 Express 直接提供 `dist/` 下的静态文件，不再启动 Vite 开发服务器。

### 5.3 使用 PM2 管理进程

```bash
# 安装 PM2
npm install -g pm2

# 启动
pm2 start dist/server.cjs --name bookvoyage

# 查看状态
pm2 status

# 设置开机自启
pm2 startup
pm2 save
```

## 6. MySQL 配置参考

### 最小配置 (my.ini / my.cnf)

```ini
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci
default-storage-engine=InnoDB

# 连接数
max_connections=100

# 日志
log_error=mysql_error.log
general_log=0
```

### 创建数据库用户 (推荐)

```sql
CREATE DATABASE IF NOT EXISTS bookvoyage
  CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

CREATE USER IF NOT EXISTS 'bookvoyage'@'localhost'
  IDENTIFIED BY 'your_secure_password';

GRANT ALL PRIVILEGES ON bookvoyage.* TO 'bookvoyage'@'localhost';
FLUSH PRIVILEGES;
```

## 7. 常见问题

### Q: 启动时提示 "MySQL connection refused"
- 确认 MySQL 服务已启动: `net start MySQL` (Windows) 或 `systemctl start mysql` (Linux)
- 检查 `.env` 中的 `DB_HOST` 和 `DB_PORT` 是否正确
- 确认防火墙未阻止 3306 端口

### Q: Gemini API 不可用
- 应用会自动降级到本地回退模式，使用内置的关键词匹配逻辑
- 如果不需要 AI 功能，可以不配置 `GEMINI_API_KEY`

### Q: 图书上传失败
- 检查 `data/books/` 目录是否存在且有写入权限
- 检查 multer 文件大小限制 (默认 50MB)
- 确认文件格式为 .txt/.epub/.pdf

### Q: 如何重置所有数据
```bash
# 删除并重建数据库
mysql -u root -p -e "DROP DATABASE IF EXISTS bookvoyage; CREATE DATABASE bookvoyage CHARACTER SET utf8mb4;"
npm run seed
```
