# BookVoyage 数据库设计文档

## 1. 数据库选型

### 选择 MySQL 8.0 的理由
- **成熟稳定**: 最广泛使用的开源关系型数据库
- **JSON 支持**: 原生 JSON 类型，适合存储 chapters 等半结构化数据
- **全文索引**: 支持中文分词搜索（可选配置）
- **事务支持**: InnoDB 引擎支持 ACID 事务
- **社区生态**: Node.js mysql2 驱动成熟，支持 prepared statements 防 SQL 注入

## 2. 数据库信息

- **数据库名**: `bookvoyage`
- **字符集**: `utf8mb4` (支持中文、emoji 等全部 Unicode 字符)
- **排序规则**: `utf8mb4_unicode_ci`
- **引擎**: `InnoDB` (支持外键、事务、行级锁)

## 3. 表结构详细说明

### 3.1 users (用户表)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 用户唯一标识 |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 用户名 |
| email | VARCHAR(100) | NOT NULL, UNIQUE | 邮箱地址 |
| password_hash | VARCHAR(255) | NOT NULL | bcrypt 哈希密码 (60字符) |
| avatar | VARCHAR(500) | DEFAULT '' | 头像 URL 或 base64 |
| bio | TEXT | DEFAULT '' | 个人简介 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 注册时间 |
| updated_at | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 最后更新时间 |

**索引**: `idx_users_email`, `idx_users_username`

### 3.2 books (图书表)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 图书唯一标识 |
| user_id | INT | FK → users.id, ON DELETE CASCADE | 所属用户 |
| title | VARCHAR(200) | NOT NULL | 书名 |
| author | VARCHAR(100) | NOT NULL | 作者 |
| country | VARCHAR(50) | DEFAULT 'Unknown' | 作者国籍 |
| cover | VARCHAR(500) | DEFAULT '' | 封面 URL/base64 |
| file_path | VARCHAR(500) | DEFAULT '' | 文件存储路径 |
| file_type | VARCHAR(10) | NOT NULL, CHECK('epub','txt','pdf') | 文件格式 |
| file_size | INT | DEFAULT 0 | 文件大小(字节) |
| category | VARCHAR(50) | DEFAULT 'Literature' | 分类/流派 |
| visibility | TINYINT(1) | DEFAULT 1 | 0=私有, 1=公开 |
| total_pages | INT | DEFAULT 0 | 总页数 |
| current_page | INT | DEFAULT 0 | 当前页码 |
| progress | INT | DEFAULT 0 | 阅读进度 0-100 |
| read_time | INT | DEFAULT 0 | 累计阅读时长(秒) |
| last_read_time | DATETIME | NULL | 最后阅读时间 |
| summary | TEXT | DEFAULT '' | 图书摘要 |
| content | LONGTEXT | DEFAULT '' | 文本内容 |
| chapters | JSON | NULL | 章节列表 `[{title, content}]` |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 导入时间 |
| updated_at | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 最后更新时间 |

**索引**: `idx_books_user`, `idx_books_visibility`, `idx_books_category`, `idx_books_country`, `idx_books_title_author`

### 3.3 reading_records (阅读记录表)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 记录唯一标识 |
| user_id | INT | FK → users.id, CASCADE | 用户 |
| book_id | INT | FK → books.id, CASCADE | 图书 |
| start_time | DATETIME | NOT NULL | 开始阅读时间 |
| end_time | DATETIME | NULL | 结束阅读时间 |
| duration | INT | DEFAULT 0 | 本次时长(秒) |
| start_page | INT | DEFAULT 0 | 开始页码 |
| end_page | INT | DEFAULT 0 | 结束页码 |
| progress | VARCHAR(100) | DEFAULT '' | 阅读位置 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**索引**: `idx_reading_user_book`, `idx_reading_book`, `idx_reading_time`

### 3.4 bookmarks (书签表)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 书签唯一标识 |
| book_id | INT | FK → books.id, CASCADE | 所属图书 |
| user_id | INT | FK → users.id, CASCADE | 创建用户 |
| position | INT | NOT NULL | 书签字符位置 |
| chapter_title | VARCHAR(200) | DEFAULT '' | 所在章节名 |
| note | TEXT | DEFAULT '' | 用户备注 |
| text_snippet | TEXT | DEFAULT '' | 书签处文本 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**索引**: `idx_bookmark_book`, `idx_bookmark_user`, `idx_bookmark_user_book`

### 3.5 reviews (书评表)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 书评唯一标识 |
| user_id | INT | FK → users.id, CASCADE | 作者 |
| book_id | INT | FK → books.id, CASCADE | 关联图书 |
| title | VARCHAR(200) | NOT NULL | 书评标题 |
| content | TEXT | NOT NULL | 书评内容 |
| score | TINYINT | NOT NULL, CHECK(1-5) | 评分 |
| longitude | DECIMAL(10,6) | NULL | 经度 |
| latitude | DECIMAL(10,6) | NULL | 纬度 |
| location_name | VARCHAR(200) | DEFAULT '' | 地理位置名称 |
| visibility | TINYINT(1) | DEFAULT 1 | 0=私有, 1=公开 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 书写时间 |
| updated_at | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**索引**: `idx_review_book`, `idx_review_user`, `idx_review_visibility`, `idx_review_score`, `idx_review_location`

### 3.6 user_sessions (会话表, 可选)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 会话唯一标识 |
| user_id | INT | FK → users.id, CASCADE | 用户 |
| token_hash | VARCHAR(64) | NOT NULL | Token SHA256 哈希 |
| expires_at | DATETIME | NOT NULL | 过期时间 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

## 5. 数据迁移策略

### 初始化脚本
- `schema.sql` — 包含 CREATE DATABASE + 所有建表语句
- `seed.ts` — Node.js 种子脚本，插入默认用户和示例数据

### 版本迁移
- 当前版本: v1.0
- 未来变更通过增量 SQL 文件管理 (如 `migrations/v1.1_add_xxx.sql`)

### 备份策略
```bash
# 导出
mysqldump -u root -p bookvoyage > backup_$(date +%Y%m%d).sql

# 恢复
mysql -u root -p bookvoyage < backup_20260710.sql
```

## 6. 存储估算

| 表 | 预估记录数 | 单行大小 | 总大小 |
|----|-----------|---------|--------|
| users | ~100 | ~1KB | ~100KB |
| books | ~1000 | ~5KB (不含content) | ~5MB |
| reading_records | ~10000 | ~200B | ~2MB |
| bookmarks | ~5000 | ~500B | ~2.5MB |
| reviews | ~2000 | ~2KB | ~4MB |
| book content (LONGTEXT) | ~1000 | ~500KB/本 | ~500MB |

预计总数据库大小: < 1GB（含图书文本内容）
