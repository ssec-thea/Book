# BookVoyage 系统架构概述

## 1. 项目简介

**BookVoyage（书旅/Marginalia）** 是一个融合图书管理、在线阅读、社区分享和地理可视化的现代化阅读平台。用户不仅可以管理个人藏书、记录阅读轨迹，还能通过交互式世界地图探索全球读者的阅读足迹。

- **产品代号**: Marginalia
- **项目类型**: 全栈 Web 应用（React SPA + Express API + MySQL）
- **核心特色**: D3.js 交互式世界地图 + 3D 书架 + AI 辅助书评生成

## 3. 技术栈对照（需求 vs 实际）

| 层次 | 需求文档指定 | 实际实现 | 说明 |
|------|-------------|---------|------|
| 前端框架 | Vue 3 | **React 19** | AI Studio 生成时选择了 React |
| 构建工具 | Vite | **Vite 6** | 一致 |
| 语言 | TypeScript | **TypeScript 5.8** | 一致 |
| UI | Element Plus/Naive UI | **Tailwind CSS v4 + 自定义** | 深色书卷风格 |
| 状态管理 | Pinia | **React useState + API** | 组件内部状态 |
| 路由 | Vue Router 4 | **Tab 切换（无路由）** | SPA 单页模式 |
| HTTP | Axios | **Fetch API** | 原生 fetch 封装 |
| 地图 | ECharts | **D3.js v7** | 更灵活的定制 |
| 阅读器 | epub.js | **自定义章节分割器** | 支持 TXT/EPUB/PDF |
| AI | 无 | **Google Gemini 3.5 Flash** | 元数据提取 + 书评生成 |
| 数据库 | 无明确指定 | **MySQL 8.0** | 6 张核心表 |
| 认证 | JWT | **JWT (HS256) + bcryptjs** | 与需求一致 |
| 后端 | 无明确指定 | **Express 4** | Node.js HTTP 框架 |

## 4. 数据流

### 4.1 认证流程
```
用户输入账号密码 → POST /api/auth/login → bcrypt验证密码
  → 生成JWT → 返回{token, user} → 前端存储token到localStorage
  → 后续请求携带Authorization: Bearer <token> → auth中间件验证
```

### 4.2 图书导入流程
```
用户选择文件 → FileReader读取 → POST /api/books (multipart)
  → multer保存文件到data/books/ → 解析EPUB/TXT元数据
  → 可选: Gemini AI提取元数据 → INSERT books表 → 返回Book对象
  → 前端更新书架状态
```

### 4.3 阅读保存流程
```
打开阅读器 → 记录startTime → 翻页阅读 → 定时保存进度(30s)
  → 退出时 POST /api/reading {bookId, progress, duration}
  → INSERT/UPDATE reading_records → 前端更新阅读统计
```

## 6. 安全设计

- **密码**: bcryptjs 10轮加盐哈希，数据库永不存明文
- **认证**: JWT HS256，7天过期，Authorization Bearer 头传递
- **SQL 注入**: 所有查询使用参数化（mysql2 prepared statements）
- **文件上传**: multer 限制文件类型和大小（最大 50MB）
- **CORS**: Express 配置允许同源请求
- **XSS**: React 默认转义 + Content-Security-Policy 头
